# Infrared Vision Deployment Package

## 목적

이 디렉터리는 적외선 카메라 기반 컴퓨터비전 서비스를 `가상화된 운영 패키지`로 설명하고 배포하기 위한 산출물을 모아둔 위치다. 현재 저장소의 로봇 실험 코드와 별개로, 현장 서비스 제안과 배포 구조를 정리한 스택 파일과 환경설정 예시를 제공한다.

## 포함 파일

- `infrared_vision_stack.compose.yml`: 서비스 가상화 스택 예시
- `infrared_vision.env.example`: 배포용 환경 변수 예시

## 스택 구성

이 스택은 아래 서비스 그룹을 기준으로 구성된다.

- `edge_gateway`: 현장 적외선 카메라 수집 및 전처리
- `inference_service`: 적외선 영상 추론
- `operations_api`: 이벤트, 사용자, 정책, 알림 관리
- `web_dashboard`: 관리자 웹 앱
- `postgres`: 이벤트 및 운영 데이터 저장
- `redis`: 실시간 캐시와 이벤트 큐
- `minio`: 캡처 이미지 및 영상 저장
- `reverse_proxy`: 외부 접근 진입점

## 사용 방식

1. `infrared_vision.env.example`를 기반으로 실제 환경 변수 파일을 작성한다.
2. 각 애플리케이션 서비스의 이미지를 CI 또는 내부 레지스트리에서 준비한다.
3. 현장 유형에 따라 물류센터 정책 또는 도로 정책을 환경 변수로 설정한다.
4. 내부망 또는 VPN 환경에서 스택을 배포한다.

## 주의사항

- 현재 저장소에는 배포용 애플리케이션 이미지가 모두 포함되어 있지 않으므로, 스택 파일은 `운영 패키지 기준서`이자 배포 템플릿 역할을 한다.
- 실운영 시에는 GPU 사용 여부, 카메라 수, 내부망 정책, 로그 보존기간에 맞춰 값을 조정해야 한다.
